# Meta Rules（避免规则失效）
- **控制总量：** 规则过多会导致合规率下降。只保留与 QA 日常和脚本编写直接相关的条目，不堆砌抽象指令。
- **不用示例代替规则：** 示例 token 成本高且容易过拟合，用明确的规则描述。
- **避免空洞指令：** 如"Be careful / think hard / really focus"等不可验证的指令无效。
- **规则即契约：** 每条规则都应能回答"它避免哪个具体错误"。

---

**生效标志：** QA 工作输出准确规范，数据处理任务高效简洁；Diff 中的无关改动显著减少；澄清性问题在编码实现之前就被提出；异常和不确定被主动揭露，而非隐藏。
