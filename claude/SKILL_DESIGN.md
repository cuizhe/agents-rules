# Skill 工程化原则

## 核心原则：Skill 必须是自包含的单元

一个可迁移复用的 skill，不应是散落的指引+外部文件，而应是一个**完整、独立、开箱即用**的工具箱。

---

## 1. 自包含封装（Self-Contained）

创建 skill 时，所有相关资源必须与 `SKILL.md` 打包在一起：

| 目录 | 用途 | 示例 |
|------|------|------|
| `scripts/` | 可执行脚本（Python、PowerShell、Shell 等） | `fill_worklog.py`、`launcher.ps1` |
| `assets/` | 数据文件、配置文件、静态资源 | `task_db.json`、`config.yaml` |
| `reference/` | 设计文档、参考模板、使用说明 | `plan.md`、`README.md` |

**禁止**在 skill 目录之外查找必需资源。若需读取数据文件，脚本必须通过相对路径（如 `os.path.dirname(__file__) + "/../assets/"`）自动定位，确保迁移后无需修改代码即可运行。

---

## 2. 零外部依赖（Zero External Dependency）

- **允许**：标准库、通用包管理器（npm/pip）中的公开依赖。
- **禁止**：依赖用户本地特定路径的文件、数据库、环境变量或预装软件（除非 skill 本身提供自动检测与引导安装）。
- **例外处理**：若必须依赖外部系统（如浏览器、特定网站），skill 应在 `SKILL.md` 中明确声明前提条件，并提供自动检测/启动脚本（如 `launcher.ps1`）。

---

## 3. 可迁移复用（Portable & Reusable）

- **路径自解析**：脚本启动时自动探测自身所在目录，据此定位 `assets/` 和 `reference/`，不假设当前工作目录。
- **配置内聚**：用户级配置（如个人偏好）与 skill 级默认配置分离。默认配置放在 `assets/` 内随 skill 分发；用户覆盖配置通过标准入口传入（如命令行参数、环境变量）。
- **单目录拷贝即可用**：将整个 skill 目录复制到任意机器、任意 Agent 环境，直接运行 `SKILL.md` 中指定的入口命令，无需额外整理文件或修改路径。

---

## 4. 文档即契约（Docs as Contract）

- `SKILL.md` 必须声明 skill 的目录结构、文件清单和运行前提。
- 所有 CLI 参数、配置项、输入输出格式，在 `SKILL.md` 中必须有明确说明和示例。
- 若 skill 需要用户手动步骤（如登录网站），`SKILL.md` 必须标注检查点，提示用户确认后再继续自动化流程。

---

## 检查清单（Checklist）

在将一个新的 skill 标记为"完成"之前，确认以下事项：

- [ ] `scripts/`、`assets/`、`reference/` 目录完整，无必需文件遗漏在外部。
- [ ] 脚本通过相对路径自动定位资源，不依赖当前工作目录。
- [ ] 删除 skill 目录外的原始文件后，skill 仍能独立运行。
- [ ] `SKILL.md` 中已更新所有命令示例，路径指向 `scripts/` 下的入口文件。
- [ ] 已列出所有外部前提条件（如浏览器、登录状态），并提供检测/自动启动脚本。
